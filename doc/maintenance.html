Maintaining an Entry in the AFP

To maintain an entry, you must have CVS access to the archive at sourceforge. To get access, register with sourceforge, and ask one of the editors to add you to the AFP project.

Setup:
Check out the archive from the CVS repository with: 

cvs -d <login>@afp.cvs.sf.net:/cvsroot/afp co -d afp-devel .
where <login> is your sourceforge login. CVS at sourceforge assumes that CVS_RSH=ssh and CVS_SERVER is unset. The command above will create a directory afp-devel where theories and additional files are located. More documentation about CVS at sourceforge is available here [afp] and here [ssh keys].

Maintenance:
Maintaining an entry means making sure that this entry works with the current Isabelle development version. Maintainers are not supposed to check in new entries. New entries must be reviewed and formally accepted. They are created on the release branch by the editors.

Depending on the type of the entry, you might want to work in close lock step with Isabelle development, i.e. fix the entry immediately each time it breaks, or loosely, i.e. only shortly before a new Isabelle release. The former is useful for libraries and base entries that are used by others, the latter is Ok for larger developments and leaf entries.

Small changes might be done by the Isabelle development team for you as the change is introduced to Isabelle (don't be surprised when your entry changes slightly over time). You will be notified when an Isabelle release nears and your entry is broken. You can also choose to receive an automatic email notification each time your entry breaks (see below).

Technicalities:

  • To get the current Isabelle development version, download and install it from the Isabelle web site.
  • For experienced users there is anonymous CVS access to the Isabelle sources. Use 
    cvs -d :pserver:anonymous@cvsbroy.informatik.tu-muenchen.de:/usr/proj/isabelle-repository/archive login
cvs -d :pserver:anonymous@cvsbroy.informatik.tu-muenchen.de:/usr/proj/isabelle-repository/archive co isabelle
    to check them out. The directory structure in the Isabelle repository is different from the distribution. It is explained in isabelle/Admin/README.repos.
  • To check if entry x works, go to afp-devel/thys/x and execute isatool make. This assumes that the command Isabelle would start up the current Isabelle development version.
  • To test all entries, or a number of them, run afp-devel/admin/testall. (Running the script with -? will show options and usage information)
  • The changes you make to the CVS repository will not show up on the AFP web pages immediately. This only happens when a new version of the archive is released (usually together with Isabelle). Please contact one of the editors if you feel there is something that should be made available immediately.

Email Notification:
You can receive an automatic email notification if entry x breaks by editing the file afp-devel/thys/x/config:

  • To switch on: add your email address to NOTIFY and cvs commit the file. NOTIFY is a space separated list. If the entry is marked as FREQUENT, the test will run daily.
  • To switch off: remove your email address from NOTIFY and cvs commit the file.
You can also choose to receive email when the status of any entry in AFP changes. This is controlled by MAIN-NOTIFY in afp-devel/admin/main-config.
